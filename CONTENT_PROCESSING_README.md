# Telephony Mastery Content Processing System

A comprehensive automated system for processing telephony mastery course content, creating public-safe truncated versions while maintaining private full content locally.

## 🎯 Purpose

This system addresses the need to:
1. **Extract first 300 words** from markdown lessons for public preview
2. **Generate AI summaries** of advanced content using Claude
3. **Maintain clean separation** between public and private content
4. **Enable safe deployment** without risk of private content leakage
5. **Automate the entire workflow** with a single command

## 🏗️ System Architecture

```
content/                          # Source markdown files
├── sip-architecture.md
├── call-transfer.md
└── ...

content-processing/               # Processed content
├── public/                       # Truncated public versions (300 words)
├── private/                      # Complete private versions (full content)
├── summaries/                    # AI-generated 5-bullet summaries
└── deploy/                       # Safe deployment package
    ├── content/                  # Public content only
    ├── summaries/                # AI summaries
    └── metadata/                 # Deployment manifests
```

## 🚀 Quick Start

### 1. Install Dependencies
```bash
pip3 install -r requirements.txt
```

### 2. Basic Processing (No AI Summaries)
```bash
./process_content.sh --skip-summaries
```

### 3. Full Processing with AI Summaries
```bash
./process_content.sh --anthropic-key "sk-ant-your-api-key"
```

### 4. Custom Word Limit
```bash
./process_content.sh --word-limit 500 --anthropic-key "sk-ant-your-api-key"
```

## 📋 Detailed Usage

### Main Processing Script
```bash
./process_content.sh [OPTIONS]

OPTIONS:
  -h, --help              Show help message
  -w, --word-limit NUM    Word limit for public content (default: 300)
  -k, --anthropic-key KEY Anthropic API key for AI summaries
  --skip-summaries        Process content without AI summaries
  --validate-only         Validate existing deployment only
  --clean                 Clean all processed content
```

### Individual Scripts

#### Content Processor
```bash
# Process all files
python3 scripts/content_processor.py --word-limit 300

# Process single file
python3 scripts/content_processor.py --single-file "sip-architecture.md" --word-limit 300
```

#### Summary Generator
```bash
# Generate all summaries
python3 scripts/summary_generator.py --anthropic-key "sk-ant-your-key"

# Generate single summary
python3 scripts/summary_generator.py --anthropic-key "sk-ant-your-key" --single-file "sip-architecture.md"
```

#### Deployment Manager
```bash
# Prepare deployment package
python3 scripts/deployment_manager.py prepare

# Validate deployment safety
python3 scripts/deployment_manager.py validate

# Check deployment status
python3 scripts/deployment_manager.py status
```

## 📁 Output Structure

### Public Content Files
Each public file contains:
- **Complete YAML frontmatter** with lesson metadata
- **First 300 words** of lesson content (preserving markdown structure)
- **Continuation notice** indicating additional content available
- **Metadata tags** marking as `content_type: truncated`

### Private Content Files  
Each private file contains:
- **Complete YAML frontmatter** with enhanced metadata
- **Full lesson content** (unchanged from source)
- **Word count statistics** for public/private splits
- **Metadata tags** marking as `content_type: complete`

### AI Summary Files
Each summary file contains:
- **5-bullet point summary** of advanced content
- **Generated by Claude 3 Haiku** for cost efficiency
- **Source file references** for traceability
- **Advanced concepts focus** beyond basic introductory material

### Deployment Package
Safe for public distribution:
- **Only truncated content** (no full lessons)
- **AI summaries only** (no original advanced content)
- **Comprehensive .gitignore** preventing private content inclusion
- **Validation checks** ensuring no data leakage

## 🔒 Security Features

### Content Separation
- **Full content never leaves private directory**

- **AI summaries generated from truncated portions only**

- **Automatic validation** prevents private content in deployment package

## 🚨 Troubleshooting

### Common Issues

**"No markdown files found"**
- Ensure you're running from the correct directory
- Check that `content/` directory exists and contains `.md` files

**"Anthropic API error"**
- Verify API key is correct and has credit
- Check network connectivity
- Ensure anthropic package is installed: `pip3 install anthropic`

**"Permission denied"**
- Make script executable: `chmod +x process_content.sh`
- Check write permissions for output directories

**"Validation failed"**
- Review validation output for specific issues
- Check for accidentally included private files
- Ensure all public files are properly marked as truncated

### Debug Mode
```bash
# Enable verbose output
export DEBUG=1
./process_content.sh --anthropic-key "your-key"
```
